Eclipse: Scrapbook Page

A Java "Scrapbook Page" in Eclipse is a specialized editor within the Eclipse Integrated Development Environment (IDE) that allows developers to quickly write, evaluate, and inspect Java code snippets without needing to create a full Java class or project for each small test.

  • Creating a Java Scrapbook Page
  1. File > New > Other...
  2. Java > Java Run/Debug > Scrapbook Page > Next
  3. Name the file and click finish

Highlight the code > Right click and choose > Execute

In a Scrapbook page:
  • You type a code fragment (not necessarily a full class).
  • You highlight what you want to run > right-click → Execute / Execute Selected Text.
  • If the code compiles and runs → output goes to the Console view.
  • If there are compile errors (e.g., missing imports, unresolved types) → Eclipse shows the red squiggly lines and error markers inside the Scrapbook document itself.
  • If there are runtime exceptions (e.g., SQL error, NullPointerException) → the stack trace is printed in the Console.
  • So yes, seeing the errors inline in the Scrapbook file is normal. It's how Eclipse tells you what's wrong before or during execution.

Don't use Scrapbook pages. Use JUnit tests instead.
