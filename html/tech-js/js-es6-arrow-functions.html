  • Arrows are a function shorthand using the => syntax which return the value of the expression.
  • Simple: 1: A function with no parameters: `() => { statements }`
  • Simple: 2: A function with 3 parameters: `(one, two, three) => { statements }`

Return Statement

In this example, the return value does not need to be defined

    isRowSelectable: () => {
        return !$scope.options.readonly;
    },

    isRowSelectable: () => {
        !$scope.options.readonly;
    },

Returning Ojbect Literals

    const row = [{rowId: -1}];

    const getCurrentUser = row => ({
        rowId: row[0].rowId,
        self: false,
        model: {
            rowId: row[0].rowId,
            fullname: null
        }
    });

    console.log(getCurrentUser(row));

  • ES6 offers some new syntax for dealing with `this`: "arrow functions".
  • The new "fat arrow" notation can be used to define anonymous functions in a simpler way.
  • Rewrite `items.forEach(function(x) {...});` to be `items.forEach((x) => {...});`
  • Single expression example: `incrementedItems = items.map((x) => x+1);`

This Keyword

  • When `this` is used inside an arrow function JavaScript uses the `this` from the outer scope.
  • Always use arrow functions with ES6 to keep the `this` keyword referring to the object instantiated from the class.
  • With arrow functions, the `this` keyword always represents the object that defined the arrow function.
